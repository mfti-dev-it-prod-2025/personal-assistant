# 1. Моя роль в проекте

В рамках командной разработки с распределением задач по функциональным модулям, я отвечал за реализацию модуля «Задачи» (Tasks).
В зону моей ответственности входила разработка системы управления задачами: проектирование модели данных, бизнес-логика, реализация REST-эндпоинтов и покрытие функциональности модуля тестами. В результате модуль был успешно интегрирован в существующее приложение с сохранением общей архитектуры.

## 2. Перечень задач и описание выполненных работ

**2.1. Анализ и проектирование**
* Проведен анализ требований к системе управления задачами
* Спроектирована модель данных для задач
* Определен перечень необходимых REST эндпоинтов
* Разработана архитектура модуля

**2.2. Разработка**

В рамках этапа разработки мной были выполнены следующие задачи:

* разработка модели данных для системы управления задачами;
* создание схем для обмена данными через API;
* реализация бизнес-логики модуля;
* реализация REST-эндпоинтов.

**2.3. Тестирование**

* разработал комплексные интеграционные тесты для каждого REST эндпоинта модуля задач.


## 3. Использованные технологии и инструменты

В ходе работы над проектом использовались следующие технологии и инструменты:

* Git, GitHub — система контроля версий, управление ветками, pull request и code review.
* Язык программирования проекта Python, фреймворк FastaAPI, библиотека SQLModel/SQLAlchemy — реализация бизнес-логики и архитектурных решений.
* Средства разработки (IDE Pycharm, линтеры, форматтеры - ruff) — для написания и проверки кода.
* Poetry — управление зависимостями проекта и виртуальным окружением.
* Инструменты командной коммуникации — обсуждение технических решений и координация работы.


## 4. Саморефлексия

* Успешно реализован модуль управления задачами с нуля до рабочего состояния
* Получен практический опыт работы с FastApi
* Получен практический опыт командной разработки

**4.2. Трудности**

* Отсутствие опыта работы с FastApi и недостаток времени на его качественное освоение
* Отсутствие опыта командной разработки

**4.3. Общие впечатления от командной работы**

Опыт командной разработки, безусловно, полезен: он позволяет перенимать знания у более опытных коллег и учиться на их подходах к решению задач. В нашей команде собрались отзывчивые люди, всегда готовые помочь, за что им огромное спасибо.

Однако на текущем этапе я считаю, что командный проект в первом семестре был преждевременным. Лично я предпочёл бы индивидуальную работу, чтобы самостоятельно пройти все этапы разработки.
